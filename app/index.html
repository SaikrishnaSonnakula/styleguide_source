About

The Web Design Styleguide (or as we call it, the styleguide) is both a styleguide document, which you can see here on this site, and a library of styling assets, like images, fonts, and stylesheets. This document was built with the same assets that are imported into responsive KP applications.

If you're interested in learning more about this concept, listen to this talk on Styleguide Driven Development by Nicole Sullivan.

Javascript UI elements

We have added a new page for Javascript UI elements, these are patterns that require the use of Javascript for specific behavior, for example, accordions and custom select menus.

Installation

The styleguide is a NPM module. You can add it as a dependency to your project by running the following command in your project directory: 

npm install -D git+ssh://git@stash.kp.org/rwd/styleguide.git

Then, include the styles in your sass file: 

@import "styleguide/modern"

Since the styleguide contains a variety of front-end assets, which all need to be handled the right way, we encourage you to use the KP Build Plugin. If you are using the Build Plugin, you're all set!

For discussion and help around including Styleguide for AEM integration, log into HipChat (with SSO credentials) and join RWD Styleguide or RWD Build Plugin.

Help

If you're having any trouble getting started, open the RWD Styleguide Hipchat room to ask for additional help with Styleguide installation, contribution, etc. If you are having problems logging into Hipchat, email Tyson Black

Responsive

For ease of development the following SASS variables are provided that contain defined breakpoints.

Usage

If you want to apply a specific style to let's say the $mobile view try this: 

.element {
  @include media($mobile) {
    background-color: black;
  }
}
Here's another example: if I want to apply $desktop-large with display but hide the view in $device: 

.element{
  @include media($desktop-large) {
    display:block;
  }
  @include media($device){
    display:none;
  }
}

Colors

Include the colors below by using the following Sass function. Pass in one of the color names below as a parameter to get its calculated hex value.

For example:


.element {
  background-color: color("robin-egg");
}

Some colors, like interactive-blue, have aliases defined as variables ($link-color) that can be used on their own without calling the colors function.

Forms

Form Controls

Always associate a label with a form field with the purpose of the field.

Base styling for checkboxes can be applied with the form-control class. These checkboxes are 20px squares.



To create checkboxes of a different size, include the following mixin in your Sass. You must provide a classname, box size, and optionally a font-size.


.my-30-px-checkbox {
  @include checkbox(30px, $medium-font-size);
}

NZ

There extra information if you click here

This is where extra content would be placed and displayed to the user when the feed item is expanded. It is up to individual teams to design the feel and spacing for this internal content in accordance with DED redlines / comps. 

var expandingItemId = $("#expandable-feed-item");
var expandingItemContent = $(".expandable-feed-item__summary-content");
var expandingItemCloseExpansion = $(".expandable-feed-item .icon-close");
var expandingItemCollapsibleContent = $(".expandable-feed-item__hidden-content");

expandingItemContent.on('click', function() {
  if(!expandingItemId.hasClass("-open")){
    expandingItemId.toggleClass("-open");
    expandingItemCollapsibleContent.toggleClass("-open");
    var expanded = expandingItemId.hasClass("-open");
    expandingItemContent.attr("aria-expanded", expanded);
    expandingItemCollapsibleContent.attr("aria-hidden", !expanded);
    expandingItemCloseExpansion.focus();
  }
});

expandingItemCloseExpansion.on('click', function() {
  expandingItemId.toggleClass("-open");
  expandingItemCollapsibleContent.toggleClass("-open");
  var expanded = expandingItemId.hasClass("-open");
  expandingItemContent.attr("aria-expanded", expanded);
  expandingItemCollapsibleContent.attr("aria-hidden", !expanded);
});

Notifications

Placeholder classes are available in _notifications.scss for the base notification styles available for Styleguide.

Errors

To create static CSS classes for error notifications you can defined them in your Sass by extending the placeholder styles.


.notification {
  &.-hard-interruption {
    @extend %hard-interruption;
  }

  &.-soft-interruption {
    @extend %soft-interruption;
  }

  &.-system-error {
    @extend %system-error;
  }

  &.-validation-error {
    @extend %validation-error;
  }
}

These classes will be processed and can be used in this fashion.

action incomplete

Use this for a WIM-hard interruption scenario

The user needs to know this action cannot be completed because of some regulation, timing or other business parameter.
action incomplete

Use this for a WIM-soft interruption scenario

The user needs to know this action cannot be completed. User may be able to continue the action later, depending on the particular scenario.
system error alert

Use this for a system error scenario

An unpredictable error has occured here and the user needs to be informed of what has happened.
input error

Use this for input/validation input errors
